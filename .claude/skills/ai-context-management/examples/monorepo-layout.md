# Example: Monorepo AGENTS.md Layout

For monorepos, place a root AGENTS.md with shared rules and per-package
AGENTS.md files for package-specific guidance. Both Claude and Copilot
auto-discover the closest AGENTS.md to the file being worked on.

```
my-monorepo/
├── AGENTS.md                          ← root: shared rules (build, lint,
│                                        repo-wide conventions, top-level
│                                        architecture)
├── CLAUDE.md                          ← root: slim addendum + @./AGENTS.md
├── packages/
│   ├── frontend/
│   │   ├── AGENTS.md                  ← package-specific: React conventions,
│   │   │                                bundler quirks, UI testing rules
│   │   └── src/
│   ├── backend/
│   │   ├── AGENTS.md                  ← package-specific: FastAPI patterns,
│   │   │                                DB migration rules, auth flows
│   │   └── src/
│   └── shared/
│       ├── AGENTS.md                  ← package-specific: type sharing,
│       │                                versioning policy
│       └── src/
└── .github/
    ├── copilot-instructions.md        ← optional; Copilot also reads root AGENTS.md natively
    └── prompts/                       ← generated by npm run skills:regenerate
        ├── audit-loop.prompt.md
        └── ...
```

## Subdirectory AGENTS.md content

Each subdirectory AGENTS.md should be **package-scoped only**:

```markdown
# AGENTS.md — packages/frontend

## Frontend Conventions

- Use Vite, not Webpack.
  Reason: 10x faster HMR; matches our Vite-first deployment pipeline.
- Components in \`src/components/\`; hooks in \`src/hooks/\`; pages in \`src/pages/\`.

## Testing

\`\`\`bash
npm test --workspace=packages/frontend
\`\`\`

## Forbidden in this package

- Direct DB queries — use the @shared/api client only.
  Reason: this package is browser-side; DB access leaks credentials.
```

## What does NOT need a sibling CLAUDE.md

Sub-package directories don't need their own CLAUDE.md. Claude's CLAUDE.md
cascade auto-discovers the root CLAUDE.md and combines it with shared content
via the `@./AGENTS.md` import. Sub-package AGENTS.md files are read by both
Claude and Copilot via their own auto-discovery rules.

## Drift detection in monorepos

`npm run context:check` checks every `(AGENTS.md, CLAUDE.md)` pair it finds,
grouped by directory. Subdirectory AGENTS.md files without sibling CLAUDE.md
are simply not paired — no drift to detect, no findings emitted.

## When to add a subdirectory AGENTS.md

Add one only when the package has **package-specific rules that don't apply at
the repo level**. Don't duplicate root rules. Examples of good triggers:

- Package uses a different stack (e.g. backend Python in an otherwise-Node repo).
- Package has unique testing or build conventions.
- Package has security boundaries (browser-side ≠ server-side).

Avoid:
- Package-level AGENTS.md that just restates root content.
- AGENTS.md with no package-specific gotchas.
