## 2. Proposed Architecture

### 2.1 `/brainstorm` Skill — Two-Layer Design (R1-H1, R1-H2)

**SKILL.md is a thin orchestration shim.** All API calls go through a Node
helper inside the repo. This matches the rest of the codebase
(`scripts/openai-audit.mjs`, `scripts/gemini-review.mjs`) and gets us:
typed responses, sentinel resolution, cost tracking, secret redaction,
testability — none of which a curl-in-shell version can give us cleanly.

**Delivery scope (R2-H1)**: v1 brainstorm runs **only when invoked from a
repo that has the synced helper bundle** (audit-loop, wine-cellar,
ai-organiser — anything that consumes `sync-to-repos.mjs`). The helper is
not standalone-installable. There is **no user-level mirror** in v1 — that
would require the helper to be on PATH or installed to a fixed home location,
which is a separate delivery problem deferred to v2 (§8).

**File layout**:

```
skills/brainstorm/SKILL.md                 # Thin orchestrator (this is what Claude reads)
scripts/brainstorm-round.mjs               # CLI entry point — takes flags + topic, emits JSON
scripts/lib/brainstorm/openai-adapter.mjs  # OpenAI provider — uses installed openai SDK
scripts/lib/brainstorm/gemini-adapter.mjs  # Gemini provider — uses @google/genai SDK
scripts/lib/brainstorm/schemas.mjs         # Zod schemas for normalized response shape
scripts/lib/brainstorm/prompt.mjs          # Single source of truth for the concept-level system prompt (R2-L1)
tests/brainstorm-round.test.mjs            # Node test runner — unit tests for adapters + redaction
```

**SKILL.md contract** (short):
1. Parse `$ARGUMENTS` → flags + topic.
2. Print "Calling providers (sentinels: openai=`latest-gpt`, gemini=`latest-pro`)…"
   — show the **sentinels requested**, not resolved IDs (Gemini-G2). Resolution
   happens inside the helper; the kickoff card cannot know the concrete IDs
   yet without an extra round-trip. The post-call rendering (Step 5) shows
   the resolved IDs from the helper's `resolvedModels` field.
3. Pass the topic via a **temp file written by Claude's `Write` tool**, then
   pipe to the helper's stdin. This bypasses Bash heredoc semantics entirely —
   no expansion risk and no delimiter-collision risk (Gemini-G1 v1+v2):
   ```
   # Claude writes the topic via the Write tool to:
   #   <repo>/.claude/tmp/brainstorm-<sid>.txt
   # Then runs:
   node scripts/brainstorm-round.mjs --topic-stdin --models openai[,gemini] [--openai-model latest-gpt] [--gemini-model latest-pro] < .claude/tmp/brainstorm-<sid>.txt
   # And cleans up:
   rm -f .claude/tmp/brainstorm-<sid>.txt
   ```
   `<sid>` is the brainstorm session ID (epoch ms or random hex). The
   temp dir `.claude/tmp/` lives **inside the repo** (gitignored — see
   §4) so file permissions are governed by the user's existing repo
   ACLs, not the world-readable `/tmp` umask (Gemini-G3). The temp file
   is removed after the helper returns.

   Heredoc is **not** used — the v5 design with `<< 'EOF_BRAINSTORM_TOPIC'`
   was rejected after Gemini's round-2 finding that any heredoc delimiter
   (random or fixed) is fragile to topic content. Write-then-pipe via a
   real file is the simpler safe path.

   Argv `--topic` remains for ad-hoc CLI use by humans, but SKILL.md always
   uses temp-file + `--topic-stdin`.
4. Read JSON stdout → render per-provider blocks per the state table (§2.4),
   and include the resolved concrete model IDs from `resolvedModels` (e.g.
   `### OpenAI (gpt-5.4)` not `### OpenAI (latest-gpt)`) — Gemini-G2.
5. Add Claude's own take.
6. Stop. Wait for user.

The concept-level system prompt is **defined only in
`scripts/lib/brainstorm/prompt.mjs`** (R2-L1). SKILL.md describes its
*intent* in human terms but does not reproduce the wording — single source
of truth. AC4 (§9a) greps the prompt module, not SKILL.md.

**`scripts/brainstorm-round.mjs` contract**:

Input flags:
| Flag | Default | Meaning |
|---|---|---|
| `--topic <text>` | required | User's topic (passed via argv, not via shell-interpolated JSON — R1-H2) |
| `--topic-stdin` | off | Alternative: read topic from stdin (for topics with awkward shell quoting) |
| `--models <csv>` | `openai` | `openai`, `gemini`, or both |
| `--openai-model <id>` | `latest-gpt` | Sentinel or concrete ID — resolved via `resolveModel()` (R1-H4) |
| `--gemini-model <id>` | `latest-pro` | Same |
| `--max-tokens <n>` | `1500` | Per-provider response cap (drives cost ceiling — R1-L10) |
| `--out <path>` | stdout | JSON output destination |

**No `--no-redact` flag (R2-H3)** — redaction is mandatory in v1. False
positives are handled by the user reviewing the rendered topic (with
`[redacted-N]` placeholders) and deciding whether to retry with a
hand-cleaned `--topic` they explicitly trust. A per-category opt-out is a
v2 concern (§8).

**Output contract is total (R2-H4)**: the helper **always emits one
schema-valid JSON document** to stdout (or `--out`), regardless of provider
outcomes. Exit codes:

| Exit | When |
|---|---|
| `0` | Helper ran. JSON document is on stdout/`--out`. Each provider has its own `state` field — caller reads those, not the exit code, to know which providers worked. |
| `1` | Argv error (bad flag, missing required), OR the helper itself produced a schema-invalid document (bug). Stderr has a one-line `code: <SLUG>` reason. |

This means SKILL.md can render deterministic provider blocks even if every
provider failed, including the misconfigured case below.

JSON output schema (Zod, in `scripts/lib/brainstorm/schemas.mjs`):

```typescript
{
  topic: string,                          // post-redaction (with `[redacted-N]` placeholders if any)
  redactionCount: number,                 // how many secret patterns were redacted
  resolvedModels: { openai?: string, gemini?: string },  // sentinel → concrete IDs
  providers: Array<{          // Order MUST match the order requested in --models (R3-M2). Out-of-order completion does not affect emit order.
    provider: 'openai' | 'gemini',
    state: 'success' | 'misconfigured' | 'timeout' | 'http_error' | 'empty' | 'malformed' | 'blocked',
    text: string | null,                  // null on non-success
    errorMessage: string | null,          // populated on non-success
    httpStatus: number | null,            // populated on http_error
    usage: { inputTokens: number, outputTokens: number } | null,
    latencyMs: number,                    // 0 for misconfigured (no call attempted)
    estimatedCostUsd: number | null
  }>,
  totalCostUsd: number
}
```

`misconfigured` (R2-H2) is set when a requested provider can't be called at
all — typically a missing API key. The helper does **not** exit 1; it skips
the call, populates `state: 'misconfigured'` with `errorMessage: 'OPENAI_API_KEY
not set'` (or similar), and continues with the other provider(s). If every
requested provider is misconfigured, exit is still 0 — the JSON document
makes the failure visible.

**Provider adapters** (one per provider, identical interface):

```typescript
async function call({ topic, model, maxTokens, signal }) → ProviderResult
```

Each adapter:
- Uses the installed SDK (`openai` 6.17, `@google/genai` 1.47 — already in `package.json`)
- Wraps the call in a 60s `AbortController` timeout
- Catches and classifies errors into the state enum above
- Returns the normalized shape — never throws to the caller

**Concept-level prompt** (single source: `scripts/lib/brainstorm/prompt.mjs`,
sent to both providers verbatim — R2-L1):
> You are a thoughtful brainstorming partner. The user is exploring an idea
> and wants your independent perspective alongside other AI models'.
> - Push back where you disagree. Don't be deferential.
> - Surface trade-offs, hidden assumptions, second-order effects.
> - Propose 1–2 concrete alternatives if you see a different path.
> - Be opinionated. 250–500 words.

(The wording above is illustrative in this plan; the canonical wording lives
in `prompt.mjs`. AC4 grep targets that file.)

**Topic discipline** (R1-H5):
- Only the user-supplied `--topic` text is sent. **No** auto-injection of
  conversation summary, file contents, repo context, AGENTS.md excerpts, or
  recent transcript. The previous draft floated "Conversation so far: …"
  auto-injection — **dropped** in v2.
- Pre-send: topic runs through `redactSecrets()` from `scripts/lib/secret-patterns.mjs`.
  Redaction is **mandatory** — there is no opt-out flag (R2-H3). If anything
  is redacted, kickoff card shows `⚠ Redacted N secret(s) before sending`
  and the JSON output exposes `redactionCount` so the user can see it
  happened. False positives are handled by the user reviewing the rendered
  topic and choosing to retry with a hand-cleaned `--topic`.
- v2 may add `--include-file <path>` for explicit file inclusion. Not in v1.

**Pre-flight checks (R2-H2)**:
- Pre-flight does **not** exit on missing API keys. Instead, each missing
  key is recorded as a `misconfigured` provider state in the JSON output;
  remaining providers run normally.
- Resolved model IDs printed in kickoff card so the user sees what was
  actually used (post-sentinel resolution).
- Argv errors (bad flag, missing required `--topic`/`--topic-stdin`) DO
  exit 1 — those are programmer errors, not runtime states.

**Malformed-payload retention (R2-M3, Gemini-G3)**:
- When a provider returns a `malformed` response, the raw payload is saved
  for debugging. Path is `path.join(repoRoot, '.claude/tmp/brainstorm-<sid>-<provider>.json')`
  — **inside the repo, not `os.tmpdir()`**. This puts file permissions
  under the user's repo ACLs instead of the world-readable `/tmp` umask
  (Gemini-G3) and guarantees Windows compatibility.
- The saved payload is run through `redactSecrets()` first — same defense
  in depth as the outbound topic.
- File written with `fs.writeFileSync(path, data, { mode: 0o600 })` for
  defense in depth even inside the repo.
- `.claude/tmp/` is added to `.gitignore` in audit-loop and synced to
  consumers (see §4 — file-level plan). The directory is created on demand
  by the helper.
- Retention: cleaned automatically by the SKILL.md (`rm -f .claude/tmp/brainstorm-<sid>.txt`
  after each call, before the synthesis prompt). Stale debug JSONs are
  user-deletable; we do not write a janitor.

For each target file:

2. If present → **skip** (idempotent — no second insertion).
3. If absent → `Edit` to insert the block. Preferred insertion point per file

     existing H2.
   - AGENTS.md: immediately after the H1 + the `> **Canonical project context...**`
     blockquote if present, else after the H1.
4. **No fuzzy fallback** — if the preferred anchor is genuinely missing
   (no H1), report that and stop. Don't improvise placement.

If we ever script this (out of scope for v1 — see §8), the script must
assert exactly one `<!-- arch-map-discoverability:start -->` per target after
running, and exit non-zero otherwise.

directory). Then `!.vscode/extensions.json` works. Apply this exact change:

```diff

+.vscode/*
+!.vscode/extensions.json
```

Also remove the duplicate `.vscode/mcp.json` at line 85 since it's redundant
under `.vscode/*`.

### 2.4 Provider-State Rendering Table (R1-M8, R2-M1)

The helper is synchronous — it returns one JSON document after all
providers settle. There is no `loading` state in the data contract (R2-M1);
SKILL.md prints a single top-level "Calling providers…" message before
invoking the helper, and the per-provider blocks appear after the helper
returns.

Each provider entry in the JSON renders one block in SKILL.md output.
State drives the block's contents:

| State | Block contents | When |
|---|---|---|
| `success` | `### <provider> (<model>)`<br>`<text verbatim>` | HTTP 200, parsed text non-empty |
| `misconfigured` | `### <provider>`<br>`⚠ Not called: <errorMessage>` (e.g. "OPENAI_API_KEY not set") | Required env var missing — provider was skipped entirely (R2-H2) |
| `timeout` | `### <provider> (<model>)`<br>`⚠ Timeout after 60s. Try again or use --max-tokens=...` | AbortController fired |
| `http_error` | `### <provider> (<model>)`<br>`⚠ HTTP <status>: <provider error message verbatim>` | Non-2xx response |
| `empty` | `### <provider> (<model>)`<br>`⚠ Empty response (provider returned no content). Possible causes: blocked by safety filter, max_tokens=0.` | HTTP 200 but no text |
| `malformed` | `### <provider> (<model>)`<br>`⚠ Malformed response — could not parse <field>. Raw payload saved to <os-tmpdir>/brainstorm-<sid>-<provider>.json.` | Schema validation failed |
| `blocked` | `### <provider> (<model>)`<br>`⚠ Blocked by safety filter: <category>` | Gemini `PROHIBITED_CONTENT` etc. |

**Partial success**: if one provider succeeds and one fails, both blocks
still render — the failure does not prevent the success from being shown.
Claude's own take always renders last regardless.

## 3. Sustainability Notes

- **Brainstorm skill model IDs auto-roll** via sentinels (`latest-gpt`,
  `latest-pro`). Quarterly STATIC_POOL refresh in the resolver covers drift
  for both. Override flags exist as escape hatches.
- **Discoverability link is durable**. `docs/architecture-map.md` path is
  baked into the architectural-memory pipeline (`scripts/symbol-index/render-mermaid.mjs`).
  HTML comment markers make future regeneration straightforward.
- **VS Code extension recommendation is non-binding**. VS Code prompts but
  doesn't force install. If the extension is renamed or replaced, update one
  line in 3 files.
- **Adapter swap-in/out**: provider adapters share an interface. Adding a
  third provider (e.g. Anthropic Haiku) is a new file under `scripts/lib/brainstorm/`,
  no orchestrator changes.
- **Brainstorm skill won't drift via sync**. SKILL.md is static markdown;
  `brainstorm-round.mjs` + adapters get added to `CORE_SCRIPTS` so they
  sync with the rest of the bundle.

## 5. Risk & Trade-off Register

| Risk | Mitigation |
|---|---|
| OpenAI/Gemini model IDs go stale | Sentinels via `resolveModel()`; user can override per-call |
| Curl call hangs | `AbortController` 60s timeout in adapters; rendered as `timeout` state |
| User invokes `/brainstorm` outside any project (no `.env`) | Adapter reads env vars only; pre-flight surfaces missing key by name |
| Discoverability link added twice on re-run | HTML comment marker idempotence |
| Existing `.vscode/extensions.json` overwritten | Read-first merge logic |
| ai-organiser gitignore negation silently fails | Use `.vscode/*` (contents) instead of `.vscode/` (dir); verify with `git check-ignore` |
| Brainstorm responses leak sensitive context | `redactSecrets()` on topic before send; `redactionCount` surfaced; topic-only (no auto file injection) |
| Cost surprise | Helper computes a **pre-call estimate** = `(input_chars/4 × input_price + max_tokens × output_price) × providers` per resolved model, prints to stderr before the API calls, and includes it in the kickoff message. Post-call `totalCostUsd` field reflects actual usage. The previous v5 design (`<max_tokens × 2> ≤ $0.05` static formula) was rejected after Gemini-G2 v2 — it ignored input cost and would mislead by 10–100× on large topic-paste inputs. |
| Synthesis happens too early (Claude jumps in) | Hard rule in SKILL.md: synthesis only on explicit user signal; trigger phrases listed |
| Adapter throws unhandled | Adapters MUST never throw to caller — catch + classify into state enum; `brainstorm-round.mjs` exits non-zero only on argv errors |
| **Shell injection via topic text containing `$(…)`, backticks, or any heredoc delimiter (Gemini-G1 v1 + v2)** | **SKILL.md MUST use the temp-file pattern** — write topic via Claude's `Write` tool to `.claude/tmp/brainstorm-<sid>.txt`, then pipe `< file` to the helper. No heredoc, no shell-string interpolation. AC24 verifies the temp-file pattern is present and forbidden patterns (heredoc, `printf '%s' "$TOPIC"`) are absent. |
| **Kickoff card hallucinates resolved model IDs (Gemini-G2)** | Pre-call kickoff card shows the **sentinel requested** (`latest-gpt`), not the resolved concrete ID. Post-call rendering uses `resolvedModels` from the JSON output. Trying to resolve in SKILL.md without invoking the helper would either guess or require a separate `--resolve-only` round-trip — both deferred to v2 (§8). |

## 6. Testing Strategy

**Unit tests** (`tests/brainstorm-round.test.mjs`, run via `npm test`):
- Argv parsing — `--models openai`, `--models openai,gemini`, `--models gemini`, missing topic, both `--topic` and `--topic-stdin` simultaneously (error)
- Sentinel resolution — `latest-gpt` resolves to a concrete ID; concrete ID passes through
- Schema validation — valid response passes; missing fields fail loudly
- Secret redaction — topic with fake `sk-...` is redacted before adapter receives it; `redactionCount` correct
- Adapter error classification — mock SDK rejection with HTTP 401 → `http_error`; mock timeout → `timeout`; mock empty content → `empty`
- **Missing key (R2-H2, R3-H2)** — `OPENAI_API_KEY` unset, `--models openai`: helper exits **0**; output JSON has `providers[0].state == 'misconfigured'` and `errorMessage` names the variable
- **All-providers-failed (R2-H4, R3-H2)** — both adapters mocked to fail (one timeout, one http_error): helper exits **0**; output JSON is schema-valid; both states present
- **Provider order (R3-M2)** — `--models gemini,openai` with both succeeding: output `providers[0].provider == 'gemini' && providers[1].provider == 'openai'`, regardless of which finishes first

## 8. Out of Scope

- **Standalone-installable brainstorm** (R2-H1) — v1 is repo-bound. To make
  brainstorm work from any cwd we'd need to either (a) publish the helper
  as an npm CLI on PATH, or (b) ship an installer that copies helper +
  node_modules to a fixed home directory. Both add packaging surface
  outside the scope of this plan. v2 task.
- **Brainstorm session memory** — not persisting brainstorm transcripts to
  Supabase. Convergence is conversational and ephemeral; durable state
  belongs in the user's hand-written follow-up plan.
- **More than 2 LLMs per round** — Claude + 2 external is already 3
  perspectives, the diminishing-returns point per audit-triad findings.
- **Cost preflight gate** — brainstorm cost is bounded by `max_tokens × 2`
  per provider, so a hard ceiling is sufficient; the audit-style preflight
  prompt is overkill for sub-$0.05 calls.
- **Per-category redaction opt-out (R2-H3)** — `--no-redact` was rejected as
  a footgun. A safer opt-out (e.g. `--allow-redaction-category aws`) with
  an explicit confirmation step is a v2 concern; v1 has no opt-out.
- **Custom system prompts per call** — single concept-level prompt covers
  the goal. v2 may add `--style <preset>`.
- **Auto file/context injection** — explicitly dropped in v2 (R1-H5). No
  auto-summary of conversation. v2 may add `--include-file <path>` with
  explicit opt-in + redaction.
- **Scripted discoverability updater** — v1 is manual via `Edit` tool with
  marker idempotence. A scripted version (with assert-one-marker post-check)
  is a future maintenance task.
- **Streaming/per-provider loading state (R2-M1)** — would require streaming
  helper output (e.g. NDJSON-per-event) and SKILL.md consuming it
  incrementally. v1 helper is synchronous; UX is "Calling providers…" then
  the assembled blocks appear together. Worth revisiting if a single
  provider call regularly exceeds 30s.
- **Structured per-provider diagnostics (R3-H3)** — extend the schema with
  `diagnosticId`, `debugArtifactPath`, `blockedCategory`, etc. so SKILL.md
  renders from typed fields instead of free-text `errorMessage`. v1
  uses free text; the rendering table in §2.4 is good enough for human
  consumption. Revisit when machine consumers (dashboards) start parsing
  the JSON.
- **Centralized model pricing module (R3-M1)** — v1 helper estimates cost
  inline using a small map of resolved model IDs → per-token prices.
  Centralising into a shared pricing module (with explicit unknown-price
  handling, rounding, and tests) is the right end state but only worth it
  once 3+ scripts need pricing. Today only `openai-audit.mjs` and (after
  this) `brainstorm-round.mjs` need it; the audit script's `modelPricing`
  approach is the prior art and brainstorm should mirror it locally.
